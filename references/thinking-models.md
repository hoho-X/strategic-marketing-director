## 1. MECE 原则

### 核心定义
```
MECE = Mutually Exclusive, Collectively Exhaustive
     = 相互独立，完全穷尽
```

### 分解检验标准
```
□ 各部分是否互不重叠？（独立性检验）
□ 加总是否覆盖全部？（完备性检验）
□ 每个层级是否同一维度？（一致性检验）
```

### 常用分解维度

| 维度 | 示例 |
|------|------|
| 时间 | Q1/Q2/Q3/Q4 |
| 空间 | 华东/华南/华北/西部 |
| 层级 | 战略/战术/执行 |
| 流程 | 认知→兴趣→购买→忠诚 |
| 结构 | 产品/价格/渠道/促销 |
| 因果 | 问题→原因→对策 |

### 营销应用
```
收入拆解：
Revenue = 流量 × 转化率 × 客单价 × 复购率

用户分层：
总用户 = 新客 + 活跃老客 + 沉睡老客 + 流失用户

营销漏斗：
曝光 → 点击 → 访问 → 咨询 → 购买 → 复购
```

## 2. 金字塔原理

### 核心结构
```
        ┌─────────────┐
        │ 核心结论    │  ← 第0层：一句话说清观点
        └─────────────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
  论点1     论点2     论点3    ← 第1层：3个支撑论点
    │         │         │
    ▼         ▼         ▼
  证据1     证据2     证据3    ← 第2层：数据/案例证据
```

### 四大原则
```
1. 结论先行：最核心的观点放在最前面
2. 以上统下：上层统领下层，下层支撑上层
3. 归类分组：同类信息归为一组
4. 逻辑递进：时间顺序/结构顺序/重要性顺序
```

### So What 测试
```
每句话问：那又怎样？
→ 直到推导出可执行的建议为止
```

### 营销报告模板
```markdown
# [核心结论]

## 为什么（3个论点）
1. 论点1 + 数据支撑
2. 论点2 + 数据支撑
3. 论点3 + 数据支撑

## 怎么做（行动建议）
- 建议1：____
- 建议2：____
- 建议3：____
```

## 3. SCQA 表达模型

### 四要素
```
S（Situation）情境：背景是什么？
C（Complication）冲突：出现了什么问题？
Q（Question）问题：核心问题是什么？
A（Answer）答案：解决方案是什么？
```

### 经典案例
```
S：我们是一家成长中的SaaS公司
C：获客成本持续上升，从$50涨到$150
Q：如何降低获客成本并保持增长？
A：实施内容营销+SEO策略，预计6个月CAC降至$80
```

### 变体应用

| 场景 | SCQA变体 |
|------|---------|
| 汇报方案 | 标准SCQA |
| 提出问题 | SCQ（暂无答案）|
| 快速说服 | 开门见山A |
| 制造悬念 | 先C后S |

## 4. 六帽思考法

### 六顶帽子 = 六种视角

| 颜色 | 视角 | 问题 | 营销应用 |
|------|------|------|---------|
| ⚪ 白帽 | 事实 | 我们知道什么数据？ | 市场数据、销售数据 |
| 🔴 红帽 | 情感 | 直觉感受是什么？ | 品牌偏好、用户情绪 |
| ⚫ 黑帽 | 风险 | 可能出什么问题？ | 竞争风险、执行风险 |
| 🟡 黄帽 | 利益 | 有什么好处？ | 机会收益、增长潜力 |
| 🟢 绿帽 | 创意 | 有什么新想法？ | 创新方案、突破策略 |
| 🔵 蓝帽 | 控制 | 下一步做什么？ | 决策汇总、行动计划 |

### 使用顺序
```
单议题分析：白→红→黑→黄→绿→蓝
风险评估：黑→黄→蓝
创意发散：绿→黄→黑→蓝
决策收敛：白→黑→黄→蓝
```

### 输出模板
```
【六帽分析】议题：____

⚪ 白帽（事实）：
- 数据1：____
- 数据2：____

🔴 红帽（情感）：
- 直觉判断：____

⚫ 黑帽（风险）：
- 风险1：____
- 风险2：____

🟡 黄帽（利益）：
- 机会1：____
- 机会2：____

🟢 绿帽（创意）：
- 创意1：____
- 创意2：____

🔵 蓝帽（结论）：
- 决策：____
- 下一步：____
```

## 5. 5W2H 分析法

### 七问框架

| 问题 | 关键点 | 营销应用 |
|------|--------|---------|
| What | 是什么？定义问题 | 产品定位、活动主题 |
| Why | 为什么？目的意义 | 战略目标、KPI设定 |
| Who | 谁？目标对象 | ICP画像、决策者 |
| When | 何时？时间节点 | 上线时间、节点规划 |
| Where | 何地？地点渠道 | 市场区域、触达渠道 |
| How | 怎么做？方法路径 | 执行策略、战术组合 |
| How Much | 多少？成本收益 | 预算、ROI预测 |

### 完整检查清单
```
□ What：具体要做什么？边界是什么？
□ Why：为什么做？不做会怎样？
□ Who：目标客户是谁？执行团队是谁？
□ When：何时启动？关键节点？
□ Where：在哪个市场？哪些渠道？
□ How：用什么方法？需要什么资源？
□ How Much：预算多少？预期收益多少？
```

## 6. 决策树

### 结构化决策框架
```
决策节点 → 机会节点 → 结果节点

         ┌─ 方案A（成功概率P1）→ 收益V1
决策问题 ─┼─ 方案B（成功概率P2）→ 收益V2
         └─ 方案C（成功概率P3）→ 收益V3

期望值计算：
E(A) = P1×V1 + (1-P1)×V1_fail
```

### 营销决策应用
```
渠道选择决策树：
投入预算 →
  ├ 搜索广告（转化率高但成本高）
  │   ├ 效果好 → 加大投入
  │   └ 效果差 → 优化关键词/暂停
  │
  ├ 社交广告（成本低但转化低）
  │   ├ 曝光好 → 内容优化
  │   └ 曝光差 → 渠道调整
  │
  └ 内容营销（长尾效应但周期长）
      ├ SEO见效 → 持续投入
      └ SEO无起色 → 调整策略
```

## 7. 脑暴思维（Brainstorming）

### 核心原则
```
1. 先发散后收敛：不批判、不否定、不完善
2. 追求数量：想法越多越好
3. 鼓励奇葩：越疯狂越好
4. 组合改进：在他人想法上叠加
```

### 结构化脑暴流程
```
第1轮：自由发散（15分钟）
→ 每人写5-10个想法

第2轮：分类整理（10分钟）
→ 按主题/维度归类

第3轮：投票筛选（5分钟）
→ 选出TOP3-5

第4轮：深化完善（10分钟）
→ 对入选方案补充细节

第5轮：可行性评估（10分钟）
→ 评估实施难度和预期效果
```

### 常用脑暴技巧
| 技巧 | 方法 |
|------|------|
| 反向思考 | 如何让问题更严重？反向就是解法 |
| 类比迁移 | 其他行业如何解决类似问题？ |
| 角色代入 | 如果我是用户/竞品/老板会怎么做？ |
| 极端假设 | 如果预算无限/为零会怎么做？ |

## 8. 因果图 / 鱼骨图

### 石川馨图结构
```
                  ┌─ 原因1.1
           ┌─ 人 ─┼─ 原因1.2
           │      └─ 原因1.3
           │
           ├─ 机 ─── 原因2.x
问题/结果 ─┤
           ├─ 料 ─── 原因3.x
           │
           ├─ 法 ─── 原因4.x
           │
           └─ 环 ─── 原因5.x
```

### 5M1E 分析维度
```
人（Man）：人员能力、态度、配置
机（Machine）：设备、工具、技术
料（Material）：原材料、信息、数据
法（Method）：流程、方法、标准
环（Environment）：环境、文化、政策
测（Measurement）：指标、标准、监控
```
