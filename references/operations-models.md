## 1. PDCA 循环

### 四步闭环
```
Plan（计划）→ Do（执行）→ Check（检查）→ Act（改进）
```

| 阶段 | 关键动作 | 营销应用 |
|------|---------|---------|
| Plan | 目标设定、方案设计 | 营销计划、预算分配 |
| Do | 执行、记录数据 | 活动/广告执行 |
| Check | 数据分析、效果评估 | ROI分析、复盘 |
| Act | 标准化、优化改进 | SOP固化、策略迭代 |

### PDCA 循环模板
```
【PDCA周期】项目：____
周期：____至____

Plan：
- 目标：____
- 策略：____
- 预算：____
- 时间：____

Do：
- 执行动作1：____（完成✅）
- 执行动作2：____（完成✅）
- 执行动作3：____（进行中）

Check：
- 核心指标达成：__%
- 偏差分析：____
- 数据洞察：____

Act：
- 复盘结论：____
- 优化措施：____
- 下周期计划：____
```

## 2. OKR 目标管理

### 核心结构
```
O（Objective）：定性目标，鼓舞人心
KR（Key Results）：定量结果，可衡量（2-5个）
```

### OKR 制定原则
```
目标（O）：
✓ 激动人心，有挑战性
✓ 可理解，无歧义
✓ 对齐上级/公司战略

关键结果（KR）：
✓ 可量化、可验证
✓ 跳一跳能达成（6/10=合理）
✓ 数量有限（最多5个）
```

### 营销 OKR 示例
```
O：成为用户首选的___品牌

KR1：品牌认知度从30%提升至50%（Q4调研）
KR2：新增用户10万，CAC控制在150元以内
KR3：付费用户月留存率从60%提升至75%
KR4：NPS从15提升至35
```

### OKR 检查清单
```
□ 目标是否足够挑战（不是KPI打包）？
□ 关键结果是否可量化？
□ 是否覆盖最重要的2-3件事？
□ 是否与上级OKR对齐？
□ 是否有明确的负责人？
```

## 3. RACI 矩阵

### 四种角色
```
R（Responsible）：执行者，具体干活
A（Accountable）：负责人，对结果负责
C（Consulted）：被咨询者，提供意见
I（Informed）：知情者，被告知结果
```

### RACI 使用规则
```
每项任务：
- 必须有且仅有1个A
- R可以有多人
- C/I非必须

团队协作：
- A负责决策，不能推诿
- R负责执行，需明确分工
- C提供专业意见，不决策
- I只被告知结果，不参与
```

### 营销项目 RACI 示例
```
【营销活动】项目：618大促

| 任务 | 市场部 | 运营部 | 产品部 | 技术部 | 销售 |
|------|--------|--------|--------|--------|------|
| 活动策划 | A | C | C | I | C |
| 内容制作 | A | R | I | I | I |
| 落地页开发 | C | C | A | R | I |
| 广告投放 | A | R | I | I | I |
| 客服培训 | C | A | I | I | R |
| 效果复盘 | A | R | C | C | C |
```

## 4. 敏捷营销

### 核心原则
```
1. 快速迭代：小步快跑、持续优化
2. 数据驱动：基于数据决策、减少主观判断
3. 跨职能协作：团队覆盖全流程
4. 实验优先：大胆假设、快速验证
```

### 敏捷营销冲刺（Sprint）
```
周期：1-2周为一个冲刺

Day 1：策划会（Sprint Planning）
- 明确本冲刺目标
- 分解任务、评估工时

Day 2-N：执行（Execution）
- 每日站会（15分钟）
- 同步进展、识别阻塞

最后一天：复盘会（Retrospective）
- 回顾成果（vs.目标）
- 识别改进点
- 下冲刺优化
```

### 敏捷看板管理
```
| 待办 | 进行中 | 已完成 | 阻塞 |
|------|--------|--------|------|
| 任务1 | 任务3 | 任务5 | - |
| 任务2 | 任务4 | 任务6 | - |
| 任务7 | - | - | - |

WIP限制：
- 进行中最多3项（避免并行）
- 阻塞项单独标注、优先解决
```

## 5. 活动 SOP 模板

### 标准操作流程结构
```
一、活动目标
- 核心目标：GMV/用户数/转化率
- 次要目标：品牌曝光/用户激活

二、时间节点
- 预热期：T-7
- 爆发期：T0
- 收尾期：T+3
- 复盘期：T+7

三、资源配置
- 人员：谁负责什么（RACI）
- 预算：各环节预算分配
- 素材：设计、文案、视频

四、执行流程
- 每日检查清单
- 关键节点确认

五、风险预案
- 风险1：流量不足 → 预案：备用渠道
- 风险2：系统崩溃 → 预案：降级方案

六、数据监控
- 实时看板
- 异常告警阈值

七、复盘模板
- 目标达成率
- 高光/低谷
- 经验教训
```

## 6. 项目管理三角

### 三要素权衡
```
范围（Scope）
    △
   ╱ ╲
  ╱   ╲
 ╱     ╲
时间 ←──→ 成本
（Time）  （Cost）
```

### 权衡原则
```
固定一要素，调整其他两要素：

1. 固定时间：
   - 时间紧 → 控制范围、增加成本
   - 示例：618 deadline固定，砍需求或加人

2. 固定成本：
   - 预算紧 → 控制范围、延长时间
   - 示例：预算固定，分批上线

3. 固定范围：
   - 需求固定 → 增加时间或成本
   - 示例：功能必做，延期或加班
```

### 营销项目应用
```
活动类型 → 三要素策略：

品牌活动（注重质量）：
- 范围优先 → 确保体验，不惜成本/时间

促销活动（注重节奏）：
- 时间优先 → 必须按时，成本/范围可调

日常运营（注重效率）：
- 成本优先 → 控制预算，范围/时间可调
```

## 7. 营销日历规划

### 年度节奏模板
```
Q1（1-3月）：春季攻势
- 春节营销（1月）
- 妇女节（3月）
- 新品发布

Q2（4-6月）：夏季爆量
- 520（5月）
- 618大促（6月）
- 暑期预热

Q3（7-9月）：秋季深耕
- 七夕（8月）
- 开学季（9月）
- 中秋国庆（9月）

Q4（10-12月）：冬季收关
- 双11（11月）
- 双12（12月）
- 年终盘点
```

### 月度执行节奏
```
第1周：上月复盘、本月规划
第2周：活动预热、内容准备
第3周：活动爆发、流量高峰
第4周：收尾、效果评估

固定动作：
- 周会：进度同步
- 月会：数据复盘
- 季度会：战略调整
```
