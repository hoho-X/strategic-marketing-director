## 1. RFM 分析模型

### 三维评分
```
R（Recency）：最近一次消费距今多久
F（Frequency）：消费频次
M（Monetary）：消费金额

评分方式：1-5分（5分最优）
```

### 用户分层矩阵

| RFM特征 | 用户类型 | 运营策略 |
|---------|---------|---------|
| R高F高M高 | 重要价值用户 | VIP服务、专属权益 |
| R高F低M高 | 重要发展用户 | 提频次、交叉销售 |
| R低F高M高 | 重要保持用户 | 召回激活、专属优惠 |
| R低F低M高 | 重要挽留用户 | 紧急召回、一对一 |
| R高F高M低 | 一般价值用户 | 提客单价、升级引导 |
| R高F低M低 | 一般发展用户 | 激活引导、新手任务 |
| R低F高M低 | 一般保持用户 | 促活运营、优惠刺激 |
| R低F低M低 | 流失用户 | 低成本触达或放弃 |

### 计算方法
```
Step 1：计算每个用户的R/F/M原始值
Step 2：按五分位分组（20%一组）
Step 3：赋分（R越小分越高，F/M越大分越高）
Step 4：组合RFM值判断用户类型

示例：
R=5分（最近7天内消费）
F=4分（月消费3-5次）
M=3分（月消费300-500元）
→ RFM=543 → 重要价值用户偏一般
```

## 2. HEART 体验度量框架

### 五维度

| 维度 | 定义 | 核心指标 |
|------|------|---------|
| Happiness（愉悦度） | 主观满意度 | NPS、CSAT、评分 |
| Engagement（参与度） | 互动深度 | DAU/MAU、使用时长 |
| Adoption（接受度） | 新功能使用 | 新功能渗透率 |
| Retention（留存率） | 持续使用 | 次日/7日/30日留存 |
| Task Success（任务完成） | 效率 | 完成率、错误率、时长 |

### 度量体系设计
```
Step 1：定义产品目标
Step 2：为每个目标选择HEART维度
Step 3：确定每个维度的核心指标
Step 4：设置信号（用户行为信号）
Step 5：设计追踪方案

示例：
目标：提高付费转化
维度：Adoption + Retention
指标：付费转化率、付费用户留存率
信号：点击付费按钮、完成支付
```

## 3. NPS 净推荐值

### 计算公式
```
NPS = 推荐者比例（9-10分）- 贬损者比例（0-6分）

范围：-100 到 +100
```

### 分层标准

| 评分 | 类型 | 占比 | 行为特征 |
|------|------|------|---------|
| 9-10 | 推荐者 | _% | 主动推荐、复购 |
| 7-8 | 中立者 | _% | 可能流失、观望 |
| 0-6 | 贬损者 | _% | 负面口碑、流失 |

### 行业基准
```
NPS > 50：优秀
NPS 30-50：良好
NPS 0-30：一般
NPS < 0：危险
```

### NPS 优化路径
```
贬损者 → 根因分析 → 系统性修复
中立者 → 体验升级 → 转化为推荐者
推荐者 → 激励推荐 → 扩大口碑效应
```

## 4. LTV-CAC 模型

### 核心公式
```
LTV（客户生命周期价值）= ARPU × 毛利率 × 平均生命周期

CAC（客户获取成本）= 营销总费用 / 新增客户数

LTV/CAC 比值评估：
> 3：健康，可持续增长
2-3：一般，需优化
1-2：危险，获客效率低
< 1：亏损，商业模式不可持续
```

### LTV 计算详解
```
方法1（简单）：
LTV = ARPU × 毛利率 × (1/流失率)

方法2（精确）：
LTV = Σ(t=1 to T) [收入t × 毛利率] / (1+折现率)^t

方法3（分层）：
按RFM分层计算不同用户群LTV
```

### 优化方向
```
LTV/CAC < 3 时的诊断：
├─ LTV偏低？
│   ├─ 提价空间？（价格弹性）
│   ├─ 复购提升？（留存运营）
│   └─ 交叉销售？（产品矩阵）
│
└─ CAC偏高？
    ├─ 渠道优化？（ROI分析）
    ├─ 转化提升？（漏斗优化）
    └─ 裂变增长？（推荐机制）
```

## 5. A/B 测试方法论

### 实验框架
```
假设 → 实验设计 → 流量分配 → 数据收集 → 统计检验 → 决策
```

### 关键参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 显著性水平 α | 假阳性概率 | 0.05 |
| 统计功效 1-β | 假阴性规避 | 0.8 |
| 最小可检测效应 MDE | 最小有意义差异 | 业务决定 |
| 样本量 | 保证统计功效 | 预先计算 |

### 测试流程
```
Step 1：明确假设
"将按钮从蓝色改为绿色，预计点击率提升10%"

Step 2：计算样本量
基于当前转化率 + MDE + α + β

Step 3：随机分流
50/50 或 90/10（灰度）

Step 4：运行实验
确保足够时长（至少1个完整周期）

Step 5：统计检验
t检验/卡方检验 → p < 0.05 则显著

Step 6：决策
显著 → 推广
不显著 → 分析原因或放弃
```

### 常见陷阱
```
□ 提前停止实验（偷看效应）
□ 多重比较不修正
□ 样本量不足导致假阴性
□ 忽略新奇效应
□ 未控制外部变量
```

## 6. GA4 数据分析框架

### 核心模型
```
事件驱动（Event-based）数据模型
用户 → 事件 → 参数
```

### 关键报告

| 报告 | 用途 | 核心指标 |
|------|------|---------|
| 实时 | 当前活跃情况 | 活跃用户、实时事件 |
| 用户获取 | 流量来源分析 | 渠道、来源、媒介 |
| 用户参与 | 行为分析 | 互动时长、事件数 |
| 转化 | 目标完成 | 转化率、收入 |
| 留存 | 用户粘性 | 留存率、回流率 |
| 漏斗 | 流程转化 | 各步转化率 |

### 营销分析维度
```
Acquisition（获取）：
- 哪个渠道带来的用户最多？
- 哪个渠道的用户质量最高？

Behavior（行为）：
- 用户在哪些页面停留最久？
- 哪些功能使用率最高？

Conversion（转化）：
- 转化漏斗在哪里断裂？
- 哪些因素影响转化？

Retention（留存）：
- 多少用户第二天回来？
- 哪些行为预示高留存？
```
